Nettoyeur de Noms de Fichiers – Version 2.0

Date : 10 mars 2026   |   Auteur : 777 Tools

1. Introduction

Le projet "Nettoyeur de Noms de Fichiers" est une application web progressive (PWA) permettant de nettoyer les noms de fichiers en supprimant automatiquement les caractères invalides (émoticônes, symboles spéciaux, ponctuation problématique, etc.) tout en conservant les extensions. L'application fonctionne entièrement côté client, sans envoi de données vers un serveur, garantissant ainsi la confidentialité des fichiers traités.

Deux modes d'utilisation sont proposés :

  • Mode Texte Saisie ou copier‑coller de listes de noms de fichiers.
  • Mode Upload Téléversement de fichiers réels (jusqu'à 500 fichiers, 2 Go chacun, 10 Go total) pour renommage et téléchargement local.

L'application intègre également un thème clair/sombre, un bouton de retour en haut, et une fonction d'archivage ZIP natif.

2. Objectifs du projet

  • Fournir un outil gratuit, simple et rapide pour nettoyer les noms de fichiers contenant des caractères non autorisés par les systèmes de fichiers.
  • Éviter les erreurs de téléchargement ou de partage dues à des noms de fichiers mal formés.
  • Garantir la confidentialité : aucun fichier n'est transmis à un serveur.
  • Offrir une interface intuitive et accessible, avec des options de personnalisation poussées.
  • Permettre le traitement par lots (batch) de centaines de fichiers.

3. Public cible

  • Utilisateurs particuliers ayant des fichiers mal nommés (photos, documents, musique).
  • Professionnels (développeurs, designers, administrateurs) manipulant de gros volumes de fichiers.
  • Toute personne souhaitant normaliser des noms de fichiers avant archivage ou partage.

4. Fonctionnalités détaillées

4.1 Mode Texte (index.html)

  • Zone de saisie : champ de texte multiligne pour coller une liste de noms (un par ligne).
  • Nettoyage : bouton "Nettoyer les Noms" appliquant les règles définies.
  • Affichage des résultats : liste des noms originaux et nettoyés, avec indication des changements.
  • Export : bouton "Exporter les Résultats" pour télécharger un fichier .txt des noms nettoyés.
  • Exemples par défaut : chargement automatique d'une liste d'exemples.

4.2 Mode Upload (upload.html)

  • Zone de glisser‑déposer (drop zone) : dépôt de fichiers ou clic pour sélectionner.
  • Limites techniques : max 500 fichiers, 2 Go/fichier, 10 Go total.
  • Barre de progression pour les gros volumes.
  • Statistiques en temps réel : nombre de fichiers, taille utilisée, fichiers nettoyés.
  • Liste des fichiers : chaque fichier affiche nom original, nom nettoyé (si traité), taille, type, boutons individuels (nettoyer, télécharger, supprimer).
  • Actions globales : "Nettoyer tous", "Télécharger tout" (avec choix du parallélisme), "Tout effacer".
  • Archivage : modal pour créer une archive ZIP (natif, sans dépendance) avec options (compression, dossier interne, historique).

4.4 Gestion des caractères invalides

  • Liste par défaut : plus de 80 caractères (émoticônes, symboles spéciaux, ponctuation, contrôles).
  • Interface de personnalisation :
    • Affichage de la liste actuelle avec défilement.
    • Bouton "Modifier la liste" ouvrant un modal.
    • Dans le modal : suppression individuelle, ajout par saisie, présélections rapides (majuscules, minuscules, chiffres, émoticônes, spéciaux, ponctuation).
    • Boutons d'exemple : en dehors du modal, groupes de caractères spéciaux ; un clic les ajoute directement.
  • Persistance : la liste personnalisée est sauvegardée dans localStorage.

4.5 Archivage (mode upload)

  • Modal accessible via un bouton dédié.
  • Options :
    • Niveau de compression (0‑9, indicatif car ZIP natif non compressé).
    • Mot de passe (désactivé, non supporté).
    • Division de l'archive (split) par taille.
    • Choix du format (ZIP actif, 7Z et RAR désactivés).
    • Nom du dossier interne dans l'archive.
  • Historique : les archives créées sont listées avec possibilité de les re-télécharger.
  • Barre de progression pendant la création.

4.6 Interface utilisateur

  • Thème clair/sombre : bouton dans la navigation, persistance via localStorage.
  • Bouton retour en haut : apparaît après défilement, remonte en douceur.
  • Responsive design : adaptation mobile (menu hamburger, empilement).
  • Accessibilité : labels ARIA, rôles, annonces pour lecteurs d'écran.
  • Notifications : toasts pour informer l'utilisateur des actions.

5. Contraintes techniques

  • Navigateurs : tous les navigateurs modernes (Chrome, Firefox, Safari, Edge).
  • Technologies : HTML5, CSS3 (Flexbox/Grid), JavaScript ES6+, localStorage, File API.
  • Performance : traitement par chunks pour les gros fichiers, Web Workers si nécessaire.
  • Hébergement : compatible avec Netlify (fichiers statiques).
  • Sécurité : aucun téléversement serveur, tout est local.

6. Équipe projet

  • Chef de projet : pilotage, validation des fonctionnalités.
  • Développeur front-end : HTML/CSS/JS, intégration des composants.
  • Développeur back-end : non requis (application 100% client).
  • Designer UI/UX : maquettes, thème, responsive.
  • Testeur QA : validation des cas d'usage, compatibilité navigateurs.
